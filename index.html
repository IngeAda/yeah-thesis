呼吸互动装置 · 沉浸体验
走到呼吸互动装置前，俯视水面屏幕。水面初始呈现混乱波纹状态。

对着麦克风，跟随旁边小卡片的提示慢慢呼气——水面波纹随呼气从中心向外扩散，越慢越完整。完成三轮后水面归于平静，出现「情绪已平复」。

这一步是整个展览体验的高潮——观众在身体上真实感受到情绪调节机制的作用，而不只是理解概念。
说明卡文案：「对着它，慢慢呼气，看水面」
全程约40—60秒，体验结束后自动重置，等待下一位观众。

2 · 呼吸互动装置（TouchDesigner 水面波纹）
核心概念与意象
借鉴吹气互动装置的表现形式，将呼吸调整机制可视化为水面波纹。情绪失控 = 水面被搅动，呼吸调节 = 让水面归于平静。观众对着麦克风慢慢呼气，触发波纹从中心向外扩散——呼气越慢越稳，波纹越完整。
呼气越慢→波纹越完整 呼气太急→波纹碎裂 三轮完成→水面归于平静
三阶段视觉状态
吸气 4秒
水面平息
暖橙→冷蓝
屏息 2秒
镜面静止
最亮最纯
呼气 6秒
波纹扩散
深蓝→浅蓝
初始水面混乱对应情绪失控状态。三轮完成后出现「情绪已平复」后淡出，等待下一位观众。全程约40—60秒。

硬件组成
主控模块
ESP32 微控制器
双核 240MHz Wi-Fi + BLE 5.0 内置 ADC/DAC
负责协调所有模块运作，通过 BLE 与手机 App 保持低功耗连接，接收三端状态指令后驱动光效与语音同步切换。
光效模块
WS2812B LED 灯环
单线串行协议 全彩 RGB 可寻址独立控制
环形排列内置于装置底座，透过磨砂亚克力漫射片形成均匀柔和的光晕效果。支持三状态颜色与亮度程序化切换。
语音模块
DFPlayer Mini + 扬声器
SD 卡存储音频 UART 串口控制 2W 功率扬声器
预存 TTS 生成的三段语音安抚内容，由 ESP32 在家长离场后延迟5秒触发，播放时与光效节律同步。
漫射与外壳
磨砂亚克力 + 3D打印壳体
磨砂漫射片 圆柱造型 内嵌走线
磨砂亚克力作为灯环漫射层，使光效均匀柔和无硬边。壳体通过 3D 打印定制，内部走线整洁，外部无裸露接口。
传感模块
HC-SR04 超声波传感器
检测距离 2—400cm 自动唤醒
展览场景下用于感应观众靠近，自动唤醒装置进入演示模式。实际家用场景中可由手机 App 替代。
供电
USB-C 5V 供电
标准 5V/2A 适配器 内置稳压模块
USB-C 接口统一供电，内置稳压芯片分别为 ESP32、LED 灯环、DFPlayer 供电，避免电流互相干扰。
软件框架
状态机逻辑（ESP32 固件）
装置内部维护三个核心状态，收到手机指令后立即切换：
STATE_ACTIVE — 辅导进行中，暖白常亮
STATE_PAUSE — 临界触发，琥珀黄慢呼吸
STATE_託管 — 家长离场，浅橙呼吸+延迟触发语音
状态切换延迟 <50ms，光效过渡采用线性插值避免闪烁。
BLE 通信协议
手机 App 通过 BLE GATT 服务向装置发送状态指令，单向下发，装置不回传数据：
Service UUID: 0xFFE0
Characteristic: 0xFFE1
CMD_ACTIVE → 0x01
CMD_PAUSE → 0x02
CMD_CUSTODY → 0x03
CMD_RESET → 0x00
连接范围约10m，低功耗模式下待机电流 <5mA。

一 · 核心概念
设计意图
将手表端的呼吸引导机制从屏幕中解放出来，放大成一个可以被多人感知、主动参与的展览装置。观众通过对着麦克风呼气，实时控制水面波纹的生成与扩散，在「让水面归于平静」的过程中，完成一次真实的情绪调节体验。

装置不教观众「如何呼吸」，而是用视觉反馈让他们自发地慢下来——这和产品本身的最小干预原则一致。
核心隐喻
情绪失控 = 水面被搅动，杂乱涟漪相互干扰
呼吸调节 = 让水面重新归于平静
一次完整呼气 = 一滴水落下，一圈干净的波纹向外扩散

波纹只在呼气阶段出现，且呼气越慢越完整——观众为了「看完这圈波纹」，会主动放慢呼气速度，过程中完成真实的呼吸调节。
二 · 视觉叙事与呼吸阶段对应

初始状态 · 观众走近前

水面呈现混乱状态——多圈杂乱涟漪相互叠加干扰，颜色偏暖橙，像未平息的情绪。感应到有人靠近后，水面开始缓缓减弱混乱，等待第一次呼气。

三 · 交互反馈机制
呼气质量决定波纹质量
装置同时运行两条数据线：
· 麦克风实时采集观众呼气响度曲线
· Timer CHOP 生成4-2-6标准节律模板

两者做相似度比对，映射到波纹参数：
呼气慢且稳 → 波纹完整同心圆 呼气太急 → 波纹碎裂不规则 呼气太短 → 波纹扩散到一半消失
视觉引导节奏
屏幕边缘显示极细的进度弧线，对应当前所在阶段（吸气/屏息/呼气），不打扰主视觉但提供节律参考。

旁边小卡片只写一句话：
「对着它，慢慢呼气，看水面」
不写节律数字，不写说明，让观众自己发现规律。

水面模拟核心逻辑
Feedback TOP 每帧把上一帧画面稍微扩散（Blur），再加轻微位移（Displace），形成自然衰减的水波。在麦克风检测到呼气触发点时，向 Feedback 注入一个高亮点，自动产生向外扩散的波纹效果——不需要手写物理模拟，TD 原生节点即可实现。
Blur 半径 → 控制波纹扩散速度 注入点亮度 → 控制波纹强度 Displace 强度 → 控制水面混乱程度

装置物理形态
横向屏幕 · 平铺桌面 尺寸建议 24—32寸 观众俯视体验

Apple Watch（家长佩戴）

A · 双通道感知（核心模块）

A-1 语气监测
麦克风采集声学特征
手表麦克风后台持续采集，不识别语音内容，仅分析声学参数
音量强度变化语速变化不识别内容
语气升级判断
音量持续超基准1.4倍 + 语速持续加快，视为语气升级早期信号

HealthKit 持续采集

A-3 双通道融合判断
任一或双重触发均可启动临界拦截
语气升级作为早期预警，心率超阈值作为确认。两者在手表端本地融合判断，无需跨端传递信号
